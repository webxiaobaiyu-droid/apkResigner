01 / OVERVIEW

加固完成，签名回归原位。

APK Resigner 将环境检测、证书选择、二次签名和结果验证收拢进一个安静的桌面工作流。它不会覆盖原文件，也不会保存你的密码。

CORE CAPABILITIES

签名需要的步骤，一个界面完成

01

自动发现环境

查找 Android SDK Build-Tools 中最新的 apksigner，并自动识别 Android Studio 或 DevEco Studio 的 JDK。

02

可靠生成新文件

选择 APK、JKS 或 Keystore 后输出全新文件，不覆盖原始 APK，也不会覆盖已有结果。

03

签名后立即验证

同时启用 V1 与 V2 签名，完成后执行详细验证并展示证书信息，失败时自动清理无效输出。

WORKFLOW

四步完成二次签名

  1. 01

    选择加固 APK

    载入本人所有或已获授权的加固后应用。

  2. 02

    选择原签名证书

    支持 JKS、Keystore 与常用证书库格式。

  3. 03

    填写证书信息

    输入 Key Alias、证书库密码和私钥密码。

  4. 04

    签名并验证

    生成新 APK，自动输出完整验证报告。

LOCAL BY DESIGN

敏感材料不离开设备

工具没有上传服务。APK、签名证书、Alias 与密码仅用于当前本地操作，密码在成功或失败后都会立即从界面清空。

仅限学习与授权场景

本项目仅供 Android 安全、应用加固与签名机制的学习研究使用。请仅处理你本人所有或已获得明确授权的应用，严禁用于篡改、盗版、绕过授权或其他违法用途。
